# Live2D 模型使用说明

## 概述

本应用现在支持运行时动态加载模型，无需重新打包应用。模型文件存储在 exe 同级的 `model/` 目录中。

## 目录结构

发布后的目录结构应该是：

```
l2d-movie-maker.exe
model/
  ├── models.json          # 模型索引文件（自动生成）
  ├── anon/                # anon 模型
  │   ├── model.json
  │   ├── texture_00.png
  │   ├── anon_casual-2023.moc
  │   └── ... (其他文件)
  └── soyo/                # soyo 模型（示例）
      ├── model.json
      ├── texture_00.png
      └── ... (其他文件)
```

## 如何添加新模型

### 方法 1：手动添加

1. 在 `model/` 目录下创建一个新文件夹（如 `soyo/`）
2. 将模型的所有文件放入该文件夹：
   - `model.json`（必需）
   - 纹理文件（.png, .jpg 等）
   - 模型文件（.moc, .moc3 等）
   - 动作文件（.mtn, .motion3.json 等）
   - 表情文件（.exp.json 等）
3. 编辑 `model/models.json`，添加新模型路径：
   ```json
   [
     "anon/model.json",
     "soyo/model.json"
   ]
   ```

### 方法 2：自动扫描（推荐）

1. 按照方法 1 的步骤 1-2 添加模型文件
2. 在应用的控制面板中点击 "🔄 刷新模型列表" 按钮
3. 应用会自动扫描 `model/` 目录并更新 `models.json`

## 模型文件要求

- 每个模型必须有一个 `model.json` 文件
- 支持 Cubism 2、3、4 格式
- 纹理文件路径在 `model.json` 中正确配置
- 动作和表情文件路径在 `model.json` 中正确配置

## 注意事项

- 模型文件夹名称会显示在模型选择下拉框中
- 确保所有相关文件都在同一个文件夹中
- 如果模型加载失败，检查文件路径和格式是否正确
- 应用启动时会自动创建 `model/` 目录（如果不存在）

## 故障排除

### 模型不显示
- 检查 `model/` 目录是否存在
- 确认 `models.json` 文件格式正确
- 点击 "刷新模型列表" 按钮重新扫描

### 模型加载失败
- 检查 `model.json` 中的文件路径是否正确
- 确认所有引用的文件都存在
- 检查文件格式是否支持

### 端口冲突
- 应用会自动选择可用端口
- 如果遇到端口问题，重启应用即可
