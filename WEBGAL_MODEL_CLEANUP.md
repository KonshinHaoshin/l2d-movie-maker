# WebGAL模式自动清理本地模型功能

## 🎯 功能概述

在WebGAL模式下成功加载模型后，系统会自动清理本地模式下的旧模型，避免模型冲突和内存占用。

## 🔧 实现原理

### 1. 模型加载流程
```
WebGAL脚本解析 → 模型加载 → 加载成功 → 清理本地模式状态 → 保持WebGAL模型显示
```

### 2. 关键函数

#### `cleanupLocalModeModelsAfterWebGAL()`
- **作用**: 在WebGAL模型加载成功后清理本地模式状态
- **特点**: 不清理当前显示的WebGAL模型，只清理本地模式相关的状态
- **清理内容**:
  - `modelData` 状态
  - `customRecordingBounds` 状态

#### `exitWebGALMode()`
- **作用**: 退出WebGAL模式时的完整清理
- **清理内容**:
  - WebGAL模型（通过`modelManager.cleanupCurrentModel()`）
  - 时间线数据
  - WebGAL模式状态

### 3. 状态管理

- **WebGAL模式激活**: 通过`importWebGALTimeline`函数标识
- **本地模式清理**: 在WebGAL模型加载成功后自动触发
- **模式切换**: 支持WebGAL模式和本地模式之间的切换

## 🚀 使用方法

### 1. 进入WebGAL模式
```typescript
// 在WebGALMode组件中输入脚本并导入
changeFigure: 改模/拼好模/大棉袄/大棉袄.jsonl -id=anon -motion=taki_smile04 -expression=soyo_smile01;
```

### 2. 自动清理流程
1. 脚本解析成功
2. 模型加载到Live2D视图
3. 自动清理本地模式状态
4. WebGAL模型保持显示

### 3. 退出WebGAL模式
- 点击"🚪 退出模式"按钮
- 系统自动清理WebGAL模型和时间线
- 返回本地模式状态

## 💡 技术特点

### 1. 智能清理
- **区分模型来源**: 只清理本地模式状态，保留WebGAL模型
- **状态隔离**: WebGAL模式和本地模式状态独立管理
- **内存优化**: 避免重复模型占用内存

### 2. 用户体验
- **无缝切换**: 支持WebGAL模式和本地模式之间的切换
- **状态保持**: 在模式内保持相关状态
- **错误处理**: 完善的错误处理和日志记录

### 3. 性能优化
- **按需清理**: 只在必要时清理相关状态
- **资源管理**: 合理管理PIXI.js资源
- **内存释放**: 及时释放不需要的模型资源

## 📝 注意事项

1. **模型路径**: 确保`figure/`目录下的模型文件路径正确
2. **端口配置**: 确保静态服务器端口配置正确（默认1431）
3. **文件格式**: 支持`.json`和`.jsonl`格式的Live2D模型
4. **状态同步**: WebGAL模式和本地模式的状态会自动同步
