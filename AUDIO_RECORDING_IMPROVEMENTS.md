# 音频录制和性能优化说明

## 🎵 音频轨道支持

### 新增功能
- **音频轨道录制**：录制时自动包含时间线中的所有音频片段
- **音频同步**：音频播放与视频录制完全同步
- **多音频混合**：支持多个音频片段同时播放和录制

### 技术实现
- 使用 `MediaElementSource` 将音频元素连接到录制流
- 通过 `MediaStreamAudioDestinationNode` 混合多个音频源
- 录制开始时自动同步启动所有音频播放

## 🎬 掉帧问题解决方案

### 问题分析
掉帧的主要原因包括：
1. **帧率过高**：60fps 对系统性能要求较高
2. **比特率过高**：高比特率增加编码负担
3. **系统性能**：CPU/GPU 性能不足
4. **浏览器限制**：某些浏览器的 MediaRecorder 实现限制

### 解决方案

#### 1. 录制质量分级
提供三种录制质量选项：

| 质量等级 | 帧率 | 比特率 | 适用场景 |
|---------|------|--------|----------|
| 低 (Low) | 24fps | 4000kbps | 性能较差的设备，快速预览 |
| 中 (Medium) | 30fps | 8000kbps | 平衡质量和性能，推荐使用 |
| 高 (High) | 60fps | 16000kbps | 高性能设备，追求最佳质量 |

#### 2. 性能优化策略
- **降低默认帧率**：从 60fps 降至 30fps
- **优化比特率**：根据质量等级调整比特率
- **音频预处理**：音频元素预加载，减少播放延迟

#### 3. 录制器优化
- **分块录制**：使用 250ms 的 timeslice 参数
- **错误处理**：增强错误处理和资源清理
- **内存管理**：及时清理音频元素和流资源

## 🚀 使用建议

### 录制质量选择
1. **首次使用**：建议选择"中"质量，平衡效果和性能
2. **性能测试**：如果录制流畅，可以尝试"高"质量
3. **性能不足**：如果出现掉帧，选择"低"质量

### 系统要求
- **低质量**：支持 WebM 编码的现代浏览器
- **中质量**：性能中等的设备，建议 4GB+ RAM
- **高质量**：性能较好的设备，建议 8GB+ RAM，独立显卡

### 最佳实践
1. **关闭其他应用**：录制时关闭不必要的应用程序
2. **避免高负载**：录制期间避免运行其他重负载任务
3. **定期测试**：在不同设备上测试录制效果

## 🔧 技术细节

### 音频轨道实现
```typescript
// 创建音频混合器
const audioDestination = ac.createMediaStreamDestination();

// 连接多个音频源
audioClips.forEach(clip => {
  const audio = new Audio(clip.audioUrl);
  const source = ac.createMediaElementSource(audio);
  source.connect(audioDestination);
});

// 添加到录制流
const audioTrack = audioDestination.stream.getAudioTracks()[0];
videoStream.addTrack(audioTrack);
```

### 质量设置配置
```typescript
const qualitySettings = {
  low: { fps: 24, kbps: 4000 },
  medium: { fps: 30, kbps: 8000 },
  high: { fps: 60, kbps: 16000 }
};
```

### 录制参数优化
```typescript
// 使用 timeslice 参数提高稳定性
mr.start(250); // 每 250ms 产出一块数据

// 根据质量选择参数
const settings = qualitySettings[recordingQuality];
createVp9AlphaRecorder(canvas, settings.fps, settings.kbps, options);
```

## 🐛 常见问题

### Q: 录制时音频不同步怎么办？
A: 确保音频文件格式兼容，尝试重新导入音频文件。

### Q: 高质量录制仍然掉帧？
A: 检查系统性能，关闭其他应用，或选择较低质量设置。

### Q: 音频轨道没有声音？
A: 检查音频文件是否损坏，确保浏览器支持音频播放。

### Q: 录制文件过大？
A: 选择较低质量设置，或减少录制时长。
